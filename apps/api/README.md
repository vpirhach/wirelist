# Wirelist API - NestJS Implementation

A modern NestJS-based API for managing wire lists with JWT authentication, built with best practices and TypeScript.

## 🚀 Features

- **NestJS Framework** - Progressive Node.js framework for building efficient and scalable applications
- **Prisma ORM** - Next-generation ORM for type-safe database access
- **JWT Authentication** - Compatible with the Java backend (shared JWT secret)
- **Role-Based Access Control** - ADMIN, USER, and VIEWER roles
- **Swagger Documentation** - Interactive API documentation
- **Pagination Support** - Efficient data retrieval with pagination
- **TypeScript** - Full type safety and modern JavaScript features
- **Validation** - Request validation using class-validator
- **PostgreSQL** - Robust relational database

## 📋 Prerequisites

- Node.js 18+ and npm/yarn
- PostgreSQL database (already set up for the Java backend)
- Existing `wirelist_db` database with tables

## 🛠️ Installation

## ⚙️ Configuration

The application uses the same database and JWT configuration as the Java backend:

**.env file:**

```env
DATABASE_URL="postgresql://postgres:mysecretpassword@localhost:5432/wirelist_db?schema=public"
PORT=3002
JWT_SECRET=404E635266556A586E3272357538782F413F4428472B4B6250645367566B5970
JWT_EXPIRATION=86400000
JWT_REFRESH_EXPIRATION=604800000
```

**Key Points:**

- `PORT=3002` - Runs on different port than Java backend (3001)
- `JWT_SECRET` - **MUST** match the Java backend for token compatibility
- Database connection uses the same `wirelist_db` database

## 🔐 Authentication

### Using Tokens from Java Backend

The JWT tokens generated by the Java backend are **100% compatible** with this Node.js backend:

1. Login via Java backend (`POST http://localhost:3001/api/auth/login`)
2. Copy the `accessToken`
3. Use it with Node.js endpoints (`Authorization: Bearer <token>`)

### Example Request

```bash
# Get all wires with pagination
curl -X GET "http://localhost:3002/wires?page=0&size=30" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"

# Filter by unit
curl -X GET "http://localhost:3002/wires?unit=UNIT1&page=0&size=30" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

## 🔌 Available Endpoints

### Wires

- `GET /wires` - Get all wires (paginated)
  - Query params: `page`, `size`, `unit`, `panel`
- `GET /wires/:id` - Get wire by ID
- `GET /wires/summary` - Get wires summary

All endpoints require JWT authentication with appropriate roles.

## 🐛 Troubleshooting

**Database connection issues:**

```bash
# Check if PostgreSQL is running
pg_isready

# Verify DATABASE_URL in .env
cat .env | grep DATABASE_URL
```

**Prisma Client not found:**

```bash
npm run prisma:generate
```

**JWT token not working:**

- Verify `JWT_SECRET` matches Java backend
- Check token expiration
- Ensure `Authorization: Bearer <token>` header is set
