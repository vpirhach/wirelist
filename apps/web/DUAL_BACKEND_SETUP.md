# Dual Backend Setup Guide

## Overview

Your Vue.js frontend now supports both Java Spring Boot (v1) and Node.js NestJS (v2) backends simultaneously!

## Configuration

### Environment Variables

Add these to your `.env` file:

```env
# API Configuration
VITE_API_BASE_URL=http://localhost:3001          # Java Spring Boot backend
VITE_API_BASE_URL_V2=http://localhost:3002       # Node.js NestJS backend
VITE_API_VERSION=v2                               # Default version (v1 or v2)
```

## Available Axios Instances

### 1. `axiosInstanceV1` - Java Spring Boot

```typescript
import { axiosInstanceV1 } from './api/axiosInstance'

// Use Java backend
const response = await axiosInstanceV1.get('/wires')
```

### 2. `axiosInstanceV2` - Node.js NestJS

```typescript
import { axiosInstanceV2 } from './api/axiosInstance'

// Use Node.js backend
const response = await axiosInstanceV2.get('/wires')
```

### 3. `axiosInstance` - Default (V2)

```typescript
import axiosInstance from './api/axiosInstance'

// Uses V2 by default
const response = await axiosInstance.get('/wires')
```

## Feature Configuration

The `API_CONFIG` object defines which backend to use for each feature:

```typescript
import { API_CONFIG } from './api/apiConfig'

// Current configuration
API_CONFIG.features = {
  auth: 'v2', // Use Node.js for authentication
  wires: 'v1', // Use Java for wires (for now)
  users: 'v2', // Use Node.js for user management
  upload: 'v1', // Use Java for file uploads (for now)
}
```

## Usage Examples

### Authentication (Node.js)

```typescript
import { authService } from './api/auth'

// All auth operations use Node.js backend
const user = await authService.login({ username: 'admin', password: 'admin123!' })
```

### Wires Management (Java)

```typescript
import { wiresService } from './api/wires'

// Uses Java backend
const wires = await wiresService.getAllWires({ page: 0, size: 30 })
```

### User Management (Node.js)

```typescript
import { userService } from './api/wires'

// Uses Node.js backend
const profile = await userService.getProfile()
```

### Dynamic Backend Selection

```typescript
import { API_CONFIG } from './api/apiConfig'

// Get the appropriate backend for a feature
const backend = API_CONFIG.features.wires === 'v2' ? axiosInstanceV2 : axiosInstanceV1
const response = await backend.get('/wires')
```

## Switching Backends

### Global Switch

Change `VITE_API_VERSION` in `.env`:

```env
# Use Node.js as default
VITE_API_VERSION=v2

# Use Java as default
VITE_API_VERSION=v1
```

### Per-Feature Switch

Update `API_CONFIG.features` in `apiConfig.ts`:

```typescript
features: {
  auth: 'v2',        // Node.js
  wires: 'v2',       // Switch to Node.js
  users: 'v2',       // Node.js
  upload: 'v1',      // Keep Java for now
}
```

## JWT Token Compatibility

✅ **Tokens work with both backends!**

- Same JWT secret key
- Same token format
- Same expiration times
- Tokens generated by one backend work with the other
